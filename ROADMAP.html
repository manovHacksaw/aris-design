Phase 4
Home Page Personalization
Authenticated users see a feed and content mosaic that reflects who they follow and their demographic profile. The most backend-heavy phase.
Duration3.5 wks
RiskHigh

Backend Tasks

TaskDetail
GET /feed/personalizedJWT required. Orchestrates steps below. Returns ranked event list
Feed step 1 — brand followsFetch all brand IDs from BrandSubscription where userId = req.user.id
Feed step 2 — creator followsFetch followed user IDs from UserFollowers. Get the brands those users own
Feed step 3 — hard filterRemove events where preferredGender ≠ user gender (unless 'All') AND where ageGroup excludes user's age (unless 'All Ages')
Feed step 4 — soft rankingScore: +2 subscribed brand, +1 domain matches vote history, +0.5 demographic soft match. Sort descending
Cold start fallbackIf result count < 10: pad with globally trending events by totalViews excluding already-included ones
GET /feed/mosaicJWT required. ?source=following|relevant&page&limit=20. Returns paginated submissions from followed creators or relevant domains
DB indexesApply before production deploy: UserFollowers(followerId), BrandSubscription(userId), Event(preferredGender), Event(ageGroup), Vote(userId, eventId)
Feed cacheIn-memory TTL cache keyed by userId. 2-minute expiry. Prevents expensive join on every navigation

7. Risk Register

RiskProbImpactMitigation
RSC conversion breaks event detail page MediumHigh Create thin RSC shell + EventDetailClient child. Test every context hook boundary. Do this first in Phase 2
AuthGuard refactor causes auth loop on edge cases MediumHigh Test all 6 auth states before merging: unauthenticated, not-onboarded, user, brand-owner, brand-pending, brand-preview-mode
Feed query slow without DB indexes HighHigh Apply indexes before any production deploy. Test with 1,000+ events in staging. Non-negotiable
IP geo unreliable (VPN, IPv6, shared IPs) HighLow Fallback to trending is immediate and silent. Geo is purely additive, never blocking
Cold start makes Home look identical to old Home HighMedium "Follow suggestions" prompt + quick-follow cards on first visit. Add domain-interest onboarding step post-launch
CSS masonry column order confuses users LowLow Document the top-to-bottom column flow. Switch to react-masonry-css with hydration suppression if feedback is negative
AI summary LLM call times out LowLow 30s timeout with "Generation failed — try again" state. Backend caches result so retry is free
Brand primaryColor field missing from schema MediumLow Check schema first. If absent, ship carousel with default blue gradient. Add color field post-launch
Feed cache TTL causes stale event visibility LowLow Document 2-minute delay as known behaviour. Not a bug. TTL is a deliberate performance trade-off

8. Recommended MVP Simplifications

The following are intentionally out of scope for this roadmap. Deferring them does not reduce product value at launch.

FeatureWhy deferredWhen to add
Short / trackable share URLsPlain event URLs are sufficient. Adds third-party service dependencyPost-launch if analytics demand it
Engagement-based soft ranking (behavioral signals)Requires new UserDomainEngagement table. Demographic-only ranking is enough for v1Personalization v2
Brand color extraction from logoCanvas pixel-sample of IPFS image. Non-trivial for marginal visual gainPhase 5 or post-launch
Onboarding interest-pickerImproves cold-start dramatically but adds onboarding scope. Use follow-suggestions prompt as interimImmediately post-Phase 4 feedback
Redis feed cacheIn-memory TTL is fine for single-instance. Add only when scaling to multiple instancesWhen horizontal scaling begins
Push notifications for personalized eventsInfrastructure exists but personalizing push content is a separate workstreamSeparate future milestone

9. What to Build First — Decision Guide

PriorityPhaseRationale
1st — AlwaysPhase 1: Foundation + AuthEvery other phase depends on the auth refactor and Explore going public. Also closes tracking gaps that affect brand analytics data quality from day one
2ndPhase 2: Explore RedesignExplore is your primary acquisition surface — public, shareable, no friction. Getting this right before deeper personalization is the correct product sequence
3rdPhase 3: Brand AnalyticsBrand retention depends on analytics. Brands need to see ROI data to keep running events. Directly impacts revenue and platform health
4thPhase 4: Home PersonalizationUser retention and depth. Pays off only after there's a critical mass of events (Phase 2) and brands running multiple events (Phase 3)
5th — Always lastPhase 5: PolishPolish compounds the value of everything before it. Never do it in the middle of a feature build
